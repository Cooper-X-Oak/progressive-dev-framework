# 渐进式开发模式

## 问题
如何在快速迭代和代码质量之间找到平衡？如何避免过早优化，同时又能积累可复用的代码和经验？

## 解决思路
采用三阶段渐进式演进：

1. **Demo 阶段** - 快速验证想法
   - 单文件实现，最小依赖
   - 不考虑复用性，只关注功能可行性
   - 快速试错，低成本放弃

2. **提炼阶段** - 系统化重构
   - 功能验证通过后再重构
   - 提取核心逻辑到 src/
   - 考虑接口设计和可维护性

3. **沉淀阶段** - 模式抽象
   - 记录解决问题的思路
   - 形成可复用的方法论
   - 建立知识库

## 适用场景
- 探索性项目，需求不明确
- 需要快速验证多个技术方案
- 希望积累可复用的解决方案
- 团队需要知识沉淀和传承

## 示例
```
场景：需要实现一个表单验证功能

1. Demo 阶段 (demo/form-validation.html)
   - 直接在 HTML 中写验证逻辑
   - 测试各种边界情况
   - 确认方案可行

2. 提炼阶段 (src/core/validator.js)
   - 提取验证逻辑为独立模块
   - 设计清晰的 API
   - 添加错误处理

3. 沉淀阶段 (patterns/form-validation-pattern.md)
   - 记录验证策略的设计思路
   - 总结常见验证场景
   - 形成可复用的模式
```

## 注意事项
- 不要跳过 Demo 阶段直接写系统化代码
- Demo 代码不需要完美，够用就行
- 只有经过验证的功能才提炼到 src/
- 模式应该是方法论，不是具体代码
- 定期回顾和更新模式库
