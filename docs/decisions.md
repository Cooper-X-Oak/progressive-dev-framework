## 决策历史

### 2026-03-12 采用渐进式开发结构
**背景**：需要一个既能快速验证想法，又能系统化沉淀的项目结构

**决策**：采用 demo → src → patterns 的三层结构

**理由**：
- demo/ 允许快速试错，不受约束
- src/ 存放验证过的系统化代码
- patterns/ 沉淀可复用的思路方法论

**影响**：
- 降低早期过度设计的风险
- 保持代码质量的同时不牺牲迭代速度
- 积累可复用的解决问题模式
