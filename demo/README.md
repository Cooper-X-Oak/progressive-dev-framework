# Demo 实验场

## 目的
快速验证功能和想法，不受系统化结构约束。

## 使用方式
1. 每个独立功能创建一个 HTML 文件
2. 可以直接在浏览器中打开测试
3. 在 `notes.md` 中记录发现和待办
4. 功能稳定后，提炼到 `src/` 目录

## 命名规范
- `feature-name.html` - 单个功能演示
- `experiment-name.html` - 实验性尝试
- 保持文件名简短、描述性强

## 示例
```html
<!-- demo/hello-world.html -->
<!DOCTYPE html>
<html>
<head>
    <title>Demo: Hello World</title>
</head>
<body>
    <h1>功能演示</h1>
    <!-- 快速验证代码 -->
</body>
</html>
```
