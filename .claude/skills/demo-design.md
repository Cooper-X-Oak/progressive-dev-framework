# demo-design

帮助用户设计 demo 的核心功能，明确验证目标。

## 执行流程

当用户调用 `/demo-design` 时，按以下步骤进行：

1. **理解需求**
   - 询问用户想要验证什么功能或想法
   - 了解背景和动机

2. **明确验证目标**
   - 这个 demo 要回答什么问题？
   - 成功的标准是什么？
   - 需要验证哪些关键点？

3. **设计最小实现**
   - 列出核心功能点（3-5个）
   - 确定技术方案（保持简单）
   - 识别可能的风险点

4. **输出 Demo 设计**
   ```markdown
   ## Demo: [功能名称]

   ### 验证目标
   - 目标1
   - 目标2

   ### 核心功能
   1. 功能点1
   2. 功能点2

   ### 技术方案
   - 使用的技术/库
   - 实现思路

   ### 验证标准
   - [ ] 标准1
   - [ ] 标准2

   ### 文件位置
   `demo/[feature-name].html`
   ```

5. **询问是否开始实现**
   - 如果用户同意，创建 demo 文件骨架
   - 在 `demo/notes.md` 中记录设计

## 原则

- 保持简单，最小化实现
- 专注于验证核心假设
- 不过度设计
- 快速迭代优于完美设计
